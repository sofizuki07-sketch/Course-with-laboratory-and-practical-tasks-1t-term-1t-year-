# Настройка переменной PATH для MSYS2 GCC в Windows

### Шаг 1: Определите правильный путь к компилятору
После установки MSYS2 и GCC через UCRT64, компилятор находится в одной из следующих папок:

- **Для UCRT64 окружения:**  
  `C:\msys64\ucrt64\bin`

- **Для MINGW64 окружения:**  
  `C:\msys64\mingw64\bin`

**Рекомендация:** Используйте путь к **UCRT64**, так как это современное окружение с поддержкой Universal C Runtime.

### Шаг 2: Откройте настройки переменных среды
1. Нажмите **Win + X** и выберите **«Система»**.
2. В правой части окна нажмите **«Дополнительные параметры системы»**.
3. В открывшемся окне нажмите кнопку **«Переменные среды»** в нижней части.

### Шаг 3: Добавьте путь в переменную PATH
1. В разделе **«Системные переменные»** найдите переменную с именем `Path`.
2. Выделите её и нажмите кнопку **«Изменить»**.
3. В открывшемся окне нажмите **«Создать»**.
4. В новую строку введите путь к папке `bin` вашего окружения MSYS2:
   - `C:\msys64\ucrt64\bin` (рекомендуется)
5. Нажмите **«OK»** во всех открытых окнах для сохранения изменений.

### Шаг 4: Проверка установки
1. Откройте **новое** окно командной строки (обязательно новое, чтобы изменения вступили в силу).
2. Введите команду:
   ```cmd
   gcc --version
   ```
3. Если всё настроено правильно, вы увидите информацию о версии GCC, например:
   ```
   gcc (GCC) 13.2.0
   Copyright (C) 2023 Free Software Foundation, Inc.
   ```

**Важные замечания:**
- Изменения в `PATH` применяются только к **новым** открытым окнам командной строки или терминала.
- Разделитель путей в Windows — точка с запятой `;`.
- Если у вас несколько версий GCC, приоритет имеет та, путь к которой указан в `PATH` раньше.
- Для работы в VSCode после изменения `PATH` необходимо перезапустить редактор.

---

### Устранение проблем
Если после настройки `gcc` всё ещё не работает:
1. Проверьте правильность пути — убедитесь, что файл `gcc.exe` действительно находится в указанной папке.
2. Убедитесь, что вы добавили путь в **системные** переменные, а не в пользовательские.
3. Попробуйте перезагрузить компьютер для применения системных изменений.
4. Проверьте, не переопределена ли переменная `PATH` в файлах конфигурации VSCode.
